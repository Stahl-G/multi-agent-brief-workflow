# MABW Runtime Agent Contract

This document is the public agent-facing contract for operating a MABW
workspace. It explains how external runtimes should use the deterministic
control plane. It does not replace the runtime handoff generated by
`multi-agent-brief run --workspace <workspace>`; the handoff remains the
per-run execution contract.

## Core Rule

Agents draft, review, and recommend. Python commands own persistent state,
stage advancement, frozen artifacts, gate reports, event logs, and delivery
archives. If an action affects a later stage, it must go through the
deterministic command that owns that state.

## Stage Order And Frozen Artifacts

The canonical stage order is:

```text
doctor
-> source-discovery
-> input-governance
-> scout
-> screener
-> claim-ledger
-> analyst
-> editor
-> auditor
-> finalize
```

The major workflow artifacts are:

| Stage | Agent-owned draft or output | Deterministic control point |
|---|---|---|
| Scout | `output/intermediate/candidate_claims.json` | `state stage-complete --stage scout` |
| Screener | `output/intermediate/screened_candidates.json` | `state stage-complete --stage screener`, unless satisfied by topology |
| Claim Ledger | `output/intermediate/claim_drafts.json` | `state freeze-claim-ledger`, then `state stage-complete --stage claim-ledger` |
| Analyst | working `output/intermediate/audited_brief.md` | Python freezes `output/intermediate/analyst_draft_snapshot.md` during Analyst completion |
| Editor | final auditable `output/intermediate/audited_brief.md` | `state stage-complete --stage editor` |
| Auditor | `output/intermediate/audit_report.json` | `gates check`, then `state stage-complete --stage auditor` |
| Formatter | `output/brief.md` and `output/delivery/*` | `state finalize-complete` creates the authoritative run archive |

Once an artifact is frozen by a completion transaction, do not edit it directly.
Use the deterministic owner-stage repair path when a repair route exists.

## Legal State Commands

Common command responsibilities:

```text
multi-agent-brief run --workspace <workspace>
  Write runtime handoff artifacts. Does not generate the brief.

multi-agent-brief state show/check --workspace <workspace>
  Read or validate control state.

multi-agent-brief state freeze-claim-ledger --workspace <workspace>
  Convert LLM-authored claim_drafts.json into Python-owned claim_ledger.json.

multi-agent-brief state stage-complete --workspace <workspace> --stage <stage>
  Complete exactly one stage through the transaction layer.

multi-agent-brief gates check --workspace <workspace>
  Write stage-scoped quality-gate reports.

multi-agent-brief repair route/start/complete --workspace <workspace>
  Route and execute deterministic owner-stage repair transactions.

multi-agent-brief finalize --workspace <workspace>
  Render reader-facing delivery.

multi-agent-brief state finalize-complete --workspace <workspace>
  Complete finalization and write the authoritative run archive.
```

Do not advance stages by editing `workflow_state.json`, `artifact_registry.json`,
or `event_log.jsonl` directly.

## Decision Boundaries

`state decide` records an operator decision; it is not a substitute for a
required completion transaction.

Use these decisions narrowly:

| Decision | Intended use | Boundary |
|---|---|---|
| `retry_stage` | Ask the current owner stage to try again before completion. | Does not re-freeze artifacts or clear gate failures by itself. |
| `delegate_repair` | Record that the current stage needs owner-stage repair routing. | Must be followed by `repair route` and, when deterministic, `repair start`; do not patch frozen artifacts directly. |
| `request_human_review` | Stop for human judgment when the deterministic route is ambiguous or a semantic decision is required. | Does not approve delivery or make a contaminated run reference-eligible. |
| `block_run` | Record that the run cannot proceed under the current evidence or control state. | Use for non-repairable integrity failures or unresolved source/support gaps. |

If a stage has required artifacts, gate blockers, claim-freeze requirements, or
finalize-complete requirements, `state decide --decision continue` cannot bypass
the corresponding deterministic command.

## Default Versus Strict Topology

MABW supports topology-aware stage satisfaction.

In default topology, Scout produces both:

```text
output/intermediate/candidate_claims.json
output/intermediate/screened_candidates.json
```

Then run:

```bash
multi-agent-brief state stage-complete --workspace <workspace> --stage scout
```

Python marks Screener complete via topology. Do not call Screener or run
`stage-complete --stage screener` again unless the handoff explicitly selected
strict topology or a repair route sends work to Screener.

In strict topology, Scout produces only `candidate_claims.json`, Screener runs
independently, and `stage-complete --stage screener` remains required.

## Quality Gate Rule Summaries

Gate reports include `rule_summary` and `docs_anchor` fields so runtime agents
can explain findings without guessing the policy.

### Quality Gate IDs

#### coverage_omission

High-priority selected screened candidates should carry into Claim Ledger claims
and, for auditable briefs, cited internal brief references. This gate checks
deterministic continuity from `screened_candidates.json` to Claim Ledger
metadata and auditable-brief `[src:<claim_id>]` references only. Reader-facing
finalize checks do not require delivery Markdown to retain internal Claim
Ledger markers. It is not a full-world recall check and does not claim the
system found every material item.

#### final_abstract_quality

Final abstract quality checks surface deterministic reader-risk patterns as
warning-only findings. The first version checks cadence/title mismatch,
comparison framing without a basis/methodology/scope section,
recommendation/forecast/superlative framing without limitations or assumptions,
incomplete key-case bullets, and locally unsupported superlatives.

This gate never blocks delivery by default, including under `--strict`. It does
not score prose, judge semantic quality, prove truth, approve delivery, decide
release readiness, open owner-stage repair, or claim that a report is
publishable.

#### material_fact

Reader-facing factual claims must be traceable to Claim Ledger support. Numbers,
material assertions, and unsupported certainty cannot be justified by prose
alone. Repair usually belongs to Editor when the brief overstates or omits a
citation, Claim Ledger when support is missing, or source-discovery when source
evidence must be refreshed.

#### freshness

Time-sensitive claims must respect the workspace report date and source
freshness window. Retrieval dates are not a substitute for publication dates
when a claim depends on publication freshness.

#### target_relevance

The executive summary and reader-facing framing must keep the configured target
entity or topic visible.

#### editor_new_fact

The Delivery Editor may polish wording, structure, and readability. The Editor
must not introduce factual tokens that were absent from the Analyst draft. If a
new fact is needed, route the repair back through the owner stages.

### Finding Types

#### target_relevance_gap

The configured target is missing from the executive summary. Repair the
reader-facing summary so the target is visible in the decision context.

#### number_without_source

A number appears in the brief without sufficient source-backed Claim Ledger
support. Add support through the proper owner stage or remove/downgrade the
number.

#### unsupported_strategic_implication

The brief uses strategic implication or recommendation language that is not
lexically supported by the frozen Claim Ledger. This is a non-blocking warning
for auditor review, not a Python proof that the implication is unsupported.
Downgrade the language, remove the implication, or add support through the
proper owner stages.

#### atomic-claim-graph

When `output/intermediate/atomic_claim_graph.json` is present and valid, it is
an optional experimental decomposition aid for frozen Claim Ledger claims. It is
not source evidence, a reader citation surface, or proof of support.

`atomic_atom_id_residue` and `atomic_graph_process_residue` findings mean
reader-facing prose exposed atom IDs or Atomic Claim Graph process wording.
Remove the residue and keep reader citations in `[src:<claim_id>]` form. Do not
create, edit, repair, or extend the graph from Analyst, Editor, or delivery
roles.

#### claim-support-matrix

When `output/intermediate/claim_support_matrix.json` is present and valid, it
records explicit atom-to-evidence-span support relations. Gate findings derived
from it are deterministic projections of those records only. They are not
Python semantic support assessment, truth proof, or release eligibility.

`claim_support_matrix_blocking_support` means a present valid matrix explicitly
records a high-materiality atom as unsupported, contradicted, or insufficiently
evidenced. `claim_support_matrix_weak_support` means the matrix records weak
support, downgrade, or adjudication needs. `claim_support_matrix_inference_framing`
means the matrix records inferential support that should be framed as inference
in reader-facing prose.

#### editor_introduced_new_fact

The Editor added factual tokens not present in
`output/intermediate/analyst_draft_snapshot.md`. Remove the new factual token or
route the intended addition through Analyst and Claim Ledger. Do not patch the
frozen snapshot.

## Repair And Contamination

If gates or audit block the run, use:

```bash
multi-agent-brief repair route --workspace <workspace> --json
multi-agent-brief repair start --workspace <workspace>
```

Delegate only the reported `repair_owner` role. Only `allowed_artifacts` may be
edited. After the owner repair, run:

```bash
multi-agent-brief repair complete --workspace <workspace> --reason "<reason>"
```

Then rerun downstream stages from `must_rerun_from`.

A contaminated run is not reference-eligible. This is evidence, not a temporary
warning to clear. A contaminated workspace may still be used for local
inspection, but clean reference evidence requires a fresh workspace or fresh run.
MABW intentionally does not provide a `reset-integrity` command.

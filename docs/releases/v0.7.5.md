# MABW v0.7.5 — Runtime Integrity And Repair Routing

MABW v0.7.5 is a control-plane hardening release. It closes several places where repeated runs, gate reports, source-discovery plans, or repair workarounds could make a run look cleaner than it really was.

The release claim stays narrow: **traceability and transaction integrity, not semantic proof yet.**

## What's New

- **Stage-scoped quality gate reports.** Auditor and finalize gates now write separate authoritative reports:
  - `output/intermediate/gates/auditor_quality_gate_report.json`
  - `output/intermediate/gates/finalize_quality_gate_report.json`

  The legacy `output/intermediate/quality_gate_report.json` remains a latest/compatibility projection, not the frozen authority for both stages.

- **Run integrity marker.** `workflow_state.json.run_integrity` records whether a run remains clean single-shot reference evidence. Resets, older-stage replay, and frozen-artifact mutation mark a run contaminated and write a matching `run_integrity_contaminated` event.

- **Immutable finalized-run archive.** Finalized runs are archived under `output/runs/<run_id>/` with delivery, intermediate, control records, and SHA-256 manifest entries so the latest `output/` surface can move forward without erasing the previous weekly run's evidence chain.

- **Python-owned audit binding.** Finalize verifies frozen Claim Ledger, audited brief, and audit report hashes through deterministic runtime control state. Auditor/LLM output is no longer responsible for writing control hash metadata.

- **Deterministic repair router.** `multi-agent-brief repair route` maps known gate, audit, registry, and workflow findings to the owning stage and allowed artifacts. It is read-only: it does not create a repair plan, mutate content, or call an agent.

- **Source-discovery hardening.** `source_candidates.yaml` is planning/review only, not evidence. Plan-only source candidates cannot be merged into `sources.yaml`, and source-discovery completion requires durable source evidence.

## Evidence

- Full deterministic test suite: 1300+ tests, no LLM calls.
- Release checks include generated-agent parity, version consistency, capability registration, public-safety scan, and release consistency.
- Codex custom-agent runtime kit remains Experimental: a real-workspace control-flow E2E reached terminal delivery, but it is not yet promoted to supported parity.

## What This Release Does Not Claim

MABW v0.7.5 does not claim output-quality improvement, autonomous repair, automatic learning, Codex parity, semantic proof, strict fact-checking, or replacement of human review.

The useful guarantee is process-level:

> When the control state says a run was clean or contaminated, the event trace must show why.

## Get Started

Use the source-clone path:

```bash
git clone https://github.com/Stahl-G/multi-agent-brief-workflow.git
cd multi-agent-brief-workflow
bash scripts/setup.sh
bash scripts/demo.sh
bash scripts/demo-deep-dive.sh
```

Then read:

- `docs/golden-path.zh-CN.md`
- `docs/weekly-use.zh-CN.md`
- `docs/launch-validation.zh-CN.md`
- `docs/support-matrix.md`

Installer assets for other distribution paths exist in the repository, but source clone remains the primary get-started path for this release.

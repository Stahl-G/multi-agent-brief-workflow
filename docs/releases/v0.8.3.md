# MABW v0.8.3 — Claim Freeze Transaction

MABW v0.8.3 moves Claim Ledger identity into deterministic Python control.
Claim Ledger agents now draft `claim_drafts.json` without claim IDs, and
`multi-agent-brief state freeze-claim-ledger` assigns deterministic `CL-####`
IDs, writes the canonical `claim_ledger.json`, records freeze metadata, and
emits a control event before the Claim Ledger stage can complete.

The goal is narrow: LLMs may do semantic claim work, but they do not mint
control IDs or freeze the ledger.

## What's New

- **Claim Draft contract.** `output/intermediate/claim_drafts.json` is an
  experimental freeze-input artifact for source-grounded claim drafts without
  `claim_id` fields. Draft entries reject `claim_id` at the top level and in
  metadata.

- **Claim Ledger freeze transaction.** `multi-agent-brief state
  freeze-claim-ledger --workspace <ws>` reads validated drafts, assigns stable
  per-freeze `CL-####` IDs, writes canonical `claim_ledger.json`, records hash
  metadata in runtime state, and emits a `claim_ledger_frozen` event.

- **Claim Ledger completion enforcement.** `state stage-complete --stage
  claim-ledger` now requires a matching freeze record for the current
  `claim_ledger.json`. Hash drift, missing freeze metadata, invalid drafts, or
  stale ledger bytes fail closed.

- **Agent contract alignment.** Claim Ledger role instructions now say to write
  `claim_drafts.json`, not `claim_ledger.json`. Analyst and Auditor contracts
  now read the frozen `claim_ledger.json`, do not read `claim_drafts.json`, and
  must not edit the ledger.

- **Auditor support calibration.** Auditor role contracts now explicitly check
  overstatement, support-strength calibration, confidence mismatch,
  evidence-relation mismatch, and limitation leakage against the frozen ledger.

## Evidence

- Full deterministic test suite runs in CI without LLM calls.
- CI includes generated-agent parity, runtime asset parity, version
  consistency, capability registration, release consistency, public-safety
  checks, and package-install smoke tests.
- Regression tests cover draft validation, `claim_id` rejection, deterministic
  freeze behavior, repeated freeze idempotency, reset-run freeze boundaries,
  missing event-log fail-closed behavior, and Analyst/Auditor contract text.

## What This Release Does Not Claim

MABW v0.8.3 does not claim semantic proof, automatic claim deduplication,
incremental claim-ID stability across edited draft sets, output-quality
improvement, autonomous repair, or Codex parity.

The `sorted_sequential_v1` claim-ID strategy is deterministic for the same
freeze input. It is not a promise that existing IDs remain stable after
incrementally adding, removing, or rewriting draft claims.

`claim_drafts.json` is a freeze input, not reader-facing evidence and not an
Analyst/Auditor input. The frozen `claim_ledger.json` remains the downstream
artifact for drafting, auditing, gates, source appendix, and finalize binding.

## Get Started

Install from source:

```bash
git clone https://github.com/Stahl-G/multi-agent-brief-workflow.git
cd multi-agent-brief-workflow
bash scripts/setup.sh
```

Create a real brief workspace:

```bash
multi-agent-brief onboard
multi-agent-brief init ~/mabw-workspace --from-onboarding onboarding.json
multi-agent-brief run --workspace ~/mabw-workspace
```

For manual or runtime-contract debugging, the Claim Ledger stage now freezes
drafts before completion:

```bash
multi-agent-brief state freeze-claim-ledger --workspace ~/mabw-workspace
multi-agent-brief state stage-complete --workspace ~/mabw-workspace --stage claim-ledger --reason "Claim Ledger frozen."
```

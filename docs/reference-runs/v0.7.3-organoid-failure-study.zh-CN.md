# v0.7.3 失败研究：一次类器官行业研究 run 暴露的边界

> Public case study. This document summarizes a private dogfooding run and the external fact-checking that followed. It does not publish the raw workspace, transcripts, full source list, local paths, private files, or unreleased operational material.

## 一句话结论

这次类器官行业研究 run 生成了一份结构完整、带来源附录、可读性较好的产业研究报告，但外部事实核查发现多处事实校准问题。它不适合作为 MABW 输出质量展示，却非常适合作为 MABW 过程问责能力的失败研究：

> 直接用大模型写简报，错误通常只留在最终文本里；用 MABW 跑简报，错误会留下传播路径。

本案例的外部核查者是 **ChatGPT 5.5 Thinking Max**。它对 MABW 产物做了公开来源核查，并在复核中同时记录了自己的误判修正。这一点很重要：外部事实核查者本身也会犯 confident-but-wrong 错误，因此 MABW 下一步要解决的不是“找一个更强模型来判真伪”，而是把 source-to-claim support calibration 做成可审计、可分级、可复盘的机制。

## 运行对象

任务是一份类器官赛道产业研究，覆盖政策监管、产业动态、科研技术、投融资与商业化方向。

MABW 产物具备完整流程表面：

- source pack；
- candidate claims；
- screened candidates；
- Claim Ledger；
- audited brief；
- quality gate report；
- finalize report；
- delivery brief；
- source appendix。

报告本身不是无来源文本。问题更细，也更接近真实业务风险：**有来源，但来源不一定支持最终 claim 的每一个限定词；事实可能部分成立，但被挂错来源、写强语气、合并进错误 claim，或被二手摘要过度认证。**

## 主要发现

### 1. FDA 段：支撑强度被写强

报告将 FDA 对 NAMs、WoE 和 fit-for-purpose 方法的监管信号写成了“类器官已被肿瘤学专项指南明确接受为非临床安全性数据来源”。

外部核查后的更稳妥表述应是：

> FDA 正在推动在特定药物类别中使用 WoE 与 fit-for-purpose NAMs 作为 supportive evidence，并探索减少部分动物毒理研究。Organoid / MPS 可作为 NAM 工具进入监管讨论，但当前公开材料不足以写成“类器官已被明确接受为非临床安全性数据来源”。

错误类型：support-strength inflation。

### 2. 中国政策段：会议新闻被升级成国家规划事实

报告把高校会议新闻中的“十五五”讨论写成了国家正式规划已经列入类器官和器官芯片技术。

更稳妥表述应是：

> 相关高校和医疗科研体系已在“十五五”语境下重点讨论类器官和器官芯片方向。是否已经进入国家正式规划文本，需要官方规划或专项文件确认。

错误类型：source authority inflation。

### 3. 欧盟 / EMA 段：真实主干和未核实子 claim 被合并

欧盟委员会动物实验淘汰路线图这一主干事实基本成立，但 EMA / OoC 的战略优先事项、反思论文和全球监管集群等子 claim 需要独立核查。报告把不同政策实体、不同来源层级和不同确定性的内容合并进一个叙述段。

错误类型：claim conflation。

### 4. 韩国案例：事实可能成立，但归因错配

技术转让、IND、I 期临床、适应症、患者数量和“全球首个”等信息被合并进一条 Claim Ledger entry，由一个技术转让来源承担过多子结论。

正确做法是拆分为多条 claim：

- 技术转让；
- IND；
- Phase 1；
- 患者数量；
- 适应症；
- “全球首个”表述。

错误类型：attribution mismatch + claim conflation。

### 5. 市场规模段：二手预测被核心化

报告引用市场规模预测作为行业判断依据，但没有核到足够强的原始报告页、方法论或发布口径。

更稳妥表述应是：

> 第三方行业媒体转述的市场研究显示，类器官市场维持高增长预期。但由于未核到原始报告页和方法论，具体市场规模数字不应作为核心判断依据。

错误类型：unverified forecast certification。

## 错误如何传播

这次失败不是单点错误，而是一个传播链：

```text
source summary
  → candidate claims
  → screened candidates
  → Claim Ledger
  → audited brief
  → final delivery
```

关键问题在 source-to-claim 之间：

- LLM / WebSearch summary 被当成了高置信 evidence；
- source entry 缺少 raw excerpt、retrieved_at、source_tier 和 excerpt hash；
- 多个子事实被压进一条 claim；
- `supported` 这类标签没有区分明确支持、方向性支持、部分支持、摘要支持和归因错配；
- freshness 规则缺少 authority 轴，新鲜二手摘要可能盖过旧的一手权威来源。

MABW v0.7.x 能追踪传播链，但不能自动证明每一步语义支撑都成立。

## MABW 做对了什么

这次 run 没有证明输出质量提升，也没有证明事实正确性。但它证明了几个过程层面的东西：

- 错误可以追溯到 source summary、candidate、Claim Ledger、audited brief 和 delivery 层。
- 问题不是消失在最终文本里，而是留下了可复盘的中间产物。
- 失败可以被归类为 support-strength inflation、authority inflation、claim conflation、attribution mismatch 和 forecast over-certification。
- 外部核查者自己的误判也可以被记录和修正，说明“更强模型”不是完整答案。

这正是 MABW 当前的主张边界：**process accountability and traceability, not semantic proof**。

## MABW 没有做对什么

这次 run 暴露出 v0.7.x 的真实边界：

- Claim Ledger 中的 source link 只说明 claim 连接到哪个 source entry，不证明原始来源逐字支持每个子主张。
- Source Appendix 是追问入口，不是事实正确性证书。
- Quality gates 能抓到格式、freshness metadata、reader residue 和部分门禁问题，但不会自动完成完整语义事实核查。
- Auditor 能按当前账本检查，但如果账本已经把 source summary 写强，Auditor 也可能忠实放大问题。

因此，这份类器官报告不能作为 MABW 输出质量 showcase，也不能作为已核验产业研究交付给外部读者。

## 已经转化成发布前修复的部分

这次失败中有三类问题属于 v0.7.3 发布前必须修的控制面问题：

1. **Audit artifact binding**

   `finalize` 必须拒绝 stale audit report：如果 audit report 仍引用当前 Claim Ledger 中不存在的 claim ID、仍带 blocking finding、仍要求 repair，或者绑定的 ledger / brief 元数据过期，终稿不得 finalize。

2. **Delivery semantics**

   README 和交付说明必须说清：交付给读者的是 `output/delivery/brief.md` 和配置好的 DOCX；source appendix 内容会附在交付稿底部，独立的 `output/source_appendix.md` 是 audit/control copy，不是额外交付文件。

3. **Public export safety**

   release consistency check 必须运行 public-safety scan，防止本地路径、token-like 字符串、环境文件引用和配置的私有词进入公开发布面。

## v0.8 方向

这次失败把 v0.8 的头号目标定义得更清楚：

- source entry 需要携带 raw excerpt、retrieved_at、original URL、source_tier、raw_excerpt_sha256、source_kind；
- 只有模型摘要、没有原文 excerpt 的 source，只能作为 discovery hint 或 summary-supported evidence；
- support relation 需要分级，例如 explicitly_supported、raw_supported、summary_supported、directionally_supported、partially_supported、supportive_but_overextended、attribution_mismatch、needs_primary_source、unsupported；
- freshness 要和 authority 双轴判断，旧的一手权威来源不应被新鲜二手摘要洗掉；
- 高风险词如“首次”“全球首个”“正式”“已获批准”“国家规划”“I 期临床”“收购”“融资金额”等，需要更强 source tier 或显式不确定性标记；
- 合取 claim 应在 Claim Ledger 阶段拆分或标记人工核查。

## 发布边界

这次失败研究可以公开，因为它说明 MABW 的边界，而不是掩盖边界。

可以公开说：

> A private organoid-industry research run produced a readable but factually overconfident brief. External review by ChatGPT 5.5 Thinking Max found that the main failure was not missing sources, but source-to-claim support calibration. MABW preserved enough intermediate artifacts to trace how the errors entered and propagated. This is exactly why v0.8 focuses on source-to-claim semantic support evaluation.

不能公开说：

> MABW proved output quality improvement.

也不能公开说：

> Every claim is proven by the Claim Ledger.

更准确的版本是：

> Every material claim links to a registered source entry: traceability, not yet semantic proof.

# BriefLoop v0.9.4 — Semantic Assessment Report Proposal Surface

v0.9.4 adds the experimental Semantic Assessment Report proposal surface.
It lets a present `output/intermediate/semantic_assessment_report.json` record
auditable semantic support assessment proposals over Claim Ledger claims,
Atomic Claim Graph atoms, and Evidence Span Registry spans.

This release does not create support truth, adjudication queues, delivery
gates, or release authority.

## What Changed

- Added optional `output/intermediate/semantic_assessment_report.json` schema
  for auditable proposal rows.
- Added machine-checkable reference validation against Claim Ledger claims,
  Atomic Claim Graph atoms, and Evidence Span Registry spans.
- Added proposal-only projection into candidate Claim-Support Matrix deltas.
  The projection is read-only and does not write accepted support rows.
- Added `status --json` and human-readable status visibility for proposal
  counts, `llm_only` rows, high uncertainty, high disagreement, and
  human-adjudication flags.
- Added public-safe synthetic dogfood fixtures covering direct support,
  partial and weak support, unsupported proposals, unknown references, and
  `llm_only` high-materiality adjudication requirements.

## Compatibility

This release does not rename:

- `multi-agent-brief` CLI
- `/mabw` Claude commands
- `multi_agent_brief` Python package/module path
- `multi-agent-brief-workflow` distribution package name
- workspace formats
- artifact names
- MABW experiment IDs such as `MABW-080`

## Experimental Status

Semantic Assessment Report support is optional and experimental:

- missing reports do not block normal workflow execution;
- invalid present reports are recorded as invalid and are not projected;
- valid reports expose proposal metadata only;
- `status` explicitly labels the surface as `proposal_only`.

## Boundaries

BriefLoop still does not prove truth, eliminate hallucinations, replace human
review, or make reports automatically ready to send.

v0.9.4 does not mutate the Claim-Support Matrix, create accepted support
truth, create adjudication queue items, add delivery gates, decide release
eligibility, or make a general output-quality claim.

# BriefLoop v0.9.3 — Evidence Span Registry And Claim-Support Matrix

v0.9.3 rolls up the experimental support-record control surfaces added after
v0.9.1: Evidence Span Registry and Claim-Support Matrix. Together with the
v0.9.1 Atomic Claim Graph foundation, this release completes the first
deterministic path from claim decomposition to span traceability to explicit
support-record projection.

This is still not semantic support assessment, truth proof, release
eligibility, or a support-sufficiency gate.

## What Changed

- Added optional `output/intermediate/evidence_span_registry.json` schema and
  runtime validation.
- Added source-pack byte binding for present Evidence Span Registry artifacts:
  `source_path` must bind to durable workspace `input/sources/` bytes, and raw
  excerpts plus optional offsets must match source text.
- Added finalized-run archive projection for valid Evidence Span Registry
  artifacts, preserving registry hashes, archived source-pack paths, source
  file hashes, source sizes, span IDs, raw-excerpt hashes, and offsets.
- Added Source Appendix span trace surfaces: reader-safe span counts in the
  appendix and raw span details only in `output/source_appendix_trace.md`.
- Added optional `output/intermediate/claim_support_matrix.json` schema for
  explicit atom-to-evidence-span support records.
- Added a deterministic policy projection helper that summarizes explicit
  matrix rows into atom-level blocking, weak-support, adjudication, downgrade,
  and inference-framing signals.
- Added cross-artifact validation for present Claim-Support Matrix artifacts
  against Claim Ledger, Atomic Claim Graph, and Evidence Span Registry records.
- Added quality-gate and status projection from present valid Claim-Support
  Matrix rows.

## Compatibility

This release does not rename:

- `multi-agent-brief` CLI
- `/mabw` Claude commands
- `multi_agent_brief` Python package/module path
- `multi-agent-brief-workflow` distribution package name
- workspace formats
- artifact names
- MABW experiment IDs such as `MABW-080`

## Experimental Status

The new support-record surfaces are optional and experimental:

- missing Evidence Span Registry and Claim-Support Matrix artifacts do not block
  normal workflow execution;
- invalid present artifacts are recorded as invalid and are not consumed for
  support projection;
- present valid Claim-Support Matrix rows can create status summaries and
  quality-gate findings, but only from explicit support records.

## Boundaries

BriefLoop still does not prove truth, eliminate hallucinations, replace human
review, or make reports automatically ready to send.

v0.9.3 does not implement semantic support assessment, automatic support-record
generation, human adjudication, release eligibility, support-sufficiency gates,
or a general output-quality claim.

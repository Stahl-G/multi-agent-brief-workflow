# Pre-Release v0.11.1 Synthetic Regression Pack

This is a public-safe reference note for the pre-release v0.11.1 synthetic
regression pack in the issue-closure line. It is not proof of output quality,
source-support sufficiency, runtime-agent compliance, delivery readiness, or
public release authorization.

## Scope

The pack adds deterministic fixtures for two Product OS / control-plane
blockers:

- durable source evidence pack manifests must reject placeholder files such as
  `input/sources/README.md` as evidence;
- release-readiness reports with forged event IDs must remain invalid in the
  artifact registry.

The cases run through `multi-agent-brief eval-cases run` using temporary
workspaces, synthetic public-safe content, and deterministic Python validators.
They do not call an LLM, fetch live sources, parse binary documents, execute a
brief workflow stage, or approve delivery.

## What This Shows

- Optional control artifacts can be present and still invalid.
- The eval-case runner can assert artifact-registry status directly, instead of
  relying only on CLI text.
- Source evidence pack materialization and release readiness remain
  inspectable control surfaces rather than reader-facing proof.

## What This Does Not Show

- It does not prove that BriefLoop writes a better brief.
- It does not prove that a source semantically supports a claim.
- It does not prove that any report is management-ready, disclosure-ready, or
  authorized for external publication.
- It does not use private customer, employer, investor-relations, or commercial
  source material.

## Reproduce

```bash
multi-agent-brief eval-cases validate
multi-agent-brief eval-cases run --json
multi-agent-brief eval-cases run --case-id source_evidence_pack_blocks_non_evidence_file --json
multi-agent-brief eval-cases run --case-id release_readiness_forged_event_blocker --json
```

Passing these cases means the deterministic blocker fixtures behaved as
expected. It is a regression signal only, not a product-quality claim.

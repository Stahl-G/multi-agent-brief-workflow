# v0.5.9 Orchestrator Contract Preparation

v0.5.9 is the privacy and architecture-status bridge before v0.6 implementation work.

The goal is to make the public repository safe and clear: public docs should describe direction and module boundaries, while detailed implementation notes remain outside tracked public files until the code is stable.

## Scope

- Keep the public roadmap high-level.
- Keep current architecture status separate from roadmap goals.
- Keep migration notes focused on the shift from Python-pipeline framing to Orchestrator-first framing.
- Keep ignored internal planning paths in place for detailed work-in-progress notes.
- Keep CI and documentation smoke checks aligned with the public/private boundary.

## Non-Goals

- No runtime behavior changes.
- No new source providers.
- No final rendering redesign.
- No full Orchestrator runtime state.
- No feedback or repair loop.
- No provenance graph implementation.
- No detailed private planning content in public docs.

## Public Acceptance

v0.5.9 is ready when:

- Public roadmap pages do not expose implementation-internal drafts or private evaluation material.
- Architecture status explains what is implemented versus future target.
- Migration notes explain that Python is the tool layer, not the standard brief-generation runtime.
- Internal planning paths are ignored.
- Documentation-only CI checks public roadmap presence and privacy boundaries without requiring private roadmap details.

## Contributor Guidance

When preparing v0.6 work, do not treat v0.5.9 as a hidden implementation release. Its value is cleanup, public clarity, and a safer place to start the Orchestrator contract work.

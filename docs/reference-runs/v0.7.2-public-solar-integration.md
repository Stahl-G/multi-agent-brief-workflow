# v0.7.2 Public Solar Integration Summary

[简体中文](v0.7.2-public-solar-integration.zh-CN.md)

This is a public-safe reference summary. It documents a two-run MABW integration run based on public solar policy and market materials. The purpose is to show whether control surfaces, Improvement Memory, and gates behaved as designed.

This is not a raw trace zip and not an A-grade causal experiment. The raw run pack stays private because it contains local paths, runtime transcripts, and debug traces. This page publishes only irreversible summaries, hash prefixes, behavior observations, and limitations.

## Verdict

Grade: **B+ integration reference**

This run demonstrated that:

- MABW can complete a two-run briefing workflow in a real runtime.
- The baseline run had no materialized Improvement Memory.
- The second run froze the human-approved `AG-0001` guidance into `improvement_memory_snapshot.md`.
- The runtime manifest recorded `materialized_entry_ids: ["AG-0001"]`.
- Quality gates actively blocked the run and later passed after repair.
- Control records existed for workflow state, artifact registry, event log, and quality gate report.

This run did not demonstrate:

- Output-quality improvement.
- Stable cross-model manifestation of guidance.
- Strict causal attribution, because run1 and run2 had different claim-layer hashes.
- An A-grade controlled reference. That would require freezing the same candidate, screened, and Claim Ledger artifacts while changing only the presence of Improvement Memory.

## Claim-Layer Hashes

This run is not a strict causal experiment because run1 and run2 had different fact-layer files:

| Artifact | run1 sha256 prefix | run2 sha256 prefix |
|---|---|---|
| `candidate_claims.json` | `b623dc598956f74e` | `3e056076a5377339` |
| `screened_candidates.json` | `34102eddcfb010f9` | `2979c14f96f9c69d` |
| `claim_ledger.json` | `cd7c4edc57213b9f` | `d67016d14195bdea` |

The correct claim is therefore: **we observed behavior changes after approved guidance materialization**, not that guidance was the only causal variable.

## Reader Output Observations

Positive behavior observed:

- run2 more often led with business implications before factual background.
- run2 expressed uncertainty and risk boundaries more explicitly.
- run2 looked more like a management brief than a plain news list.

Problems exposed:

- The Flash output at the time leaked reader-facing internal claim IDs and process wording.
- That failure directly motivated the v0.7.2 reader-final gate and delivery hardening.
- For that reason, this page does not publish the raw reader brief as showcase output.

## Public-Safety Handling

Public content keeps only:

- control-plane conclusions;
- hash prefixes;
- materialization evidence summaries;
- run statistics without local paths;
- generalized guidance text without private company facts.

Public content excludes:

- raw transcripts;
- local absolute paths;
- private operator paths or usernames;
- the wrong-model raw pack;
- the original reader brief that contained reader-output residue.

## How To Cite This

Safe wording:

> A public-safe solar integration run before v0.7.2 demonstrated Improvement Memory materialization, active gate enforcement, and complete control records in a real runtime run.

Do not say:

> MABW proved output-quality improvement.

Do not say:

> MABW proved approved guidance has a causal effect on the brief.

The accurate boundary is:

> This is an integration reference, not a controlled causal evaluation. A-grade manifestation experiments are deferred to the v0.8 evaluation track.

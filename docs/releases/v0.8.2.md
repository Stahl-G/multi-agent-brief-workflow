# MABW v0.8.2 — Role Topology And Runtime-State Safety

MABW v0.8.2 is a role-topology and control-surface release. It shortens the
default role assignment by letting Scout perform discovery and screening, while
keeping the accountable artifacts separate: candidate claims, screened
candidates, Claim Ledger, gate reports, audit report, event log, archive, and
human-triggered delivery remain part of the spine.

The release also finishes a runtime-state safety cleanup that makes the
deterministic control layer easier to audit before later topology and evaluation
work.

## What's New

- **Role topology selector.** Policy packs can select `default`, `strict`, or
  `human_assisted` topology. The default topology lets Scout write both
  `candidate_claims.json` and `screened_candidates.json`; strict topology keeps
  Screener independent.

- **Topology-satisfied stage recording.** When a stage is satisfied by topology,
  MABW records `stage_satisfied_by_topology` in the event log and workflow state
  instead of fabricating a downstream stage-complete event.

- **Topology-aware runtime handoff.** Runtime handoff and role assets now explain
  default and strict Scout/Screener behavior. The stage-completion protocol says
  when Screener is satisfied by Scout and when strict topology requires
  independent screening.

- **Human status topology line.** Human `status` output now includes compact
  topology lines such as `screener complete via scout`, using the existing
  timing projection. JSON schema, event log, workflow state, and artifact
  contracts are unchanged.

- **Editor-new-fact quality gate.** Stage-scoped gates now include a
  soft-by-default lexical check for editor-introduced numbers, claim references,
  and simple entity phrases, backed by a Python-written Analyst draft snapshot.
  `--strict` can make these findings blocking.

- **Runtime-state safety extraction.** Runtime-state helpers are split behind a
  pinned facade into manifest/workflow, artifact registry, event log, completion
  gates, and operations modules. Structural tests guard the main safety surfaces
  against reintroducing hidden helper/proxy paths.

- **Packaged handoff smoke.** CI now validates package-installed `init` and
  `run --workspace` handoff behavior for default topology and a strict-topology
  contract-base override.

## Evidence

- Full deterministic test suite runs in CI without LLM calls.
- CI includes generated-agent parity, runtime asset parity, version
  consistency, capability registration, release consistency, and public-safety
  checks.
- Package-installed topology handoff smoke verifies both default and strict
  topology handoff text/protocol outside editable source-clone mode.

## What This Release Does Not Claim

MABW v0.8.2 does not claim faster measured runtime, output-quality improvement,
semantic proof, autonomous repair, or Codex parity.

Role topology controls role assignment, not artifact accountability. Default
topology removes the default independent Screener delegation, but it does not
remove screened candidates, Claim Ledger, gates, audit report, event log,
archive, or human delivery.

`editor_new_fact` is a deterministic lexical detector. It can flag obvious
editor-introduced factual tokens, but it is not proof that every editor change
is semantically supported.

Packaged topology smoke does not package source-clone runtime kit directories
such as `.claude`, `.codex`, `.opencode`, or `.agents`. Packaged runtime-kit
distribution remains future v0.9 work.

## Get Started

Install from source:

```bash
git clone https://github.com/Stahl-G/multi-agent-brief-workflow.git
cd multi-agent-brief-workflow
bash scripts/setup.sh
```

Create a real brief workspace:

```bash
multi-agent-brief onboard
multi-agent-brief init ~/mabw-workspace --from-onboarding onboarding.json
multi-agent-brief run --workspace ~/mabw-workspace
```

Use strict topology when you want Screener to remain an independent role; use
the default topology when Scout should discover and screen while still writing
both accountable artifacts.

# BriefLoop v0.9.0 Preview — Brand Rename And Public Framing

v0.9.0 introduces **BriefLoop** as the public project-facing name.

Formerly **MABW — Multi-Agent Brief Workflow**.

## What Changed

- README and public documentation now use BriefLoop as the public project name.
- MABW is retained as the implementation lineage and compatibility surface.
- `docs/briefloop-naming.md` defines the naming and compatibility policy.
- `docs/brief-loop-engineering.md` defines brief-loop engineering.
- The public v0.9 roadmap now points toward support sufficiency and
  brief-loop engineering instead of distribution/reference workflows.

## Compatibility

This preview does not rename:

- `multi-agent-brief` CLI
- `/mabw` Claude commands
- `multi_agent_brief` Python package/module path
- `multi-agent-brief-workflow` distribution package name
- workspace formats
- artifact names
- reference-run paths
- MABW experiment IDs such as `MABW-080`

## v0.9 Direction

The minimum v0.9 support-sufficiency path is:

```text
Atomic Claim Graph -> Evidence Span Registry -> Claim-Support Matrix
```

Later v0.9.x candidates include semantic assessment proposals, human
adjudication, coverage/omission gates, semantic regression, release
eligibility, quality packs, and finding-to-repair workflows.

## Boundaries

BriefLoop does not prove truth, eliminate hallucinations, replace human review,
or make reports automatically ready to send.

v0.9.0 is a brand/public-framing preview. It does not implement Atomic Claim
Graph, Evidence Span Registry, Claim-Support Matrix, or formal A-controlled
output-quality claims.
